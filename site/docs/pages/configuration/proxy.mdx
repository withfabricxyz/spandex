# Proxy Mode

Use proxy mode when your meta-aggregation layer runs as a service and clients fetch quotes from
that service, instead of calling provider APIs directly from the client.

## When to Use It

- You want provider API keys and provider-selection logic on the server.
- You want a single policy point for rate limits, allowlists, and request validation.
- You run multi-region services and want region-aware provider configuration.

## Client Configuration

```ts
import { proxy } from "@spandex/core";
import { SpandexProvider } from "@spandex/react";

<SpandexProvider
  config={{
    proxy: proxy({ pathOrUrl: "/api/quotes" }),
  }}
>
  {children}
</SpandexProvider>;
```

With `proxy` configured, quote fetching is delegated to your service endpoint.

## Delegated Actions

`delegatedActions` controls which top-level quote APIs are delegated to your proxy service.

```ts
import { createConfig, proxy } from "@spandex/core";

const config = createConfig({
  proxy: proxy({
    pathOrUrl: "https://example.com/api/v1",
    delegatedActions: [], // only prepareQuotes is delegated
  }),
});
```

By default, no extra actions are delegated:

```ts
proxy({
  pathOrUrl: "https://example.com/api/v1",
  // delegatedActions defaults to []
});
```

When `getQuote` or `getQuotes` is delegated, simulation runs on the server. This lets your backend
own provider and RPC configuration (plus retries, limits, and policy controls) in one place.

If you set `delegatedActions: []`, only `prepareQuotes` is delegated.

## Choosing a Delegation Strategy

| `delegatedActions` | What gets delegated to proxy |
| --- | --- |
| `[]` (default) | `prepareQuotes` only |
| `["getQuotes"]` | `prepareQuotes`, `getQuotes` |
| `["getQuote"]` | `prepareQuotes`, `getQuote` |
| `["getQuotes", "getQuote"]` | `prepareQuotes`, `getQuotes`, `getQuote` |

<Callout type="info">
  Delegated <code>getQuote</code> only supports built-in strategy names on the server side. Custom
  strategy functions require <code>getQuote</code> to remain non-delegated.
</Callout>

## How It Works

1. Client calls quote APIs/hooks (`getQuotes`, `getQuote`, `getRawQuotes`, `useQuotes`, etc).
2. `AggregatorProxy` serializes swap params into query params and calls your endpoint.
3. `prepareQuotes` uses a streaming response (`newQuoteStream(...)`) so quotes arrive progressively.
4. `delegatedActions` decides whether `getQuotes`/`getQuote` are also handled entirely on the
   service.
5. For streamed flows, client decodes with `decodeQuoteStream(...)` and resolves each quote
   incrementally.

Related APIs:

- [`newQuoteStream`](/core/functions/newQuoteStream)
- [`decodeQuoteStream`](/core/functions/decodeQuoteStream)

## Hono Service Example

This pattern is implemented in `examples/hono/src/index.ts`.

```ts
import { Hono } from "hono";
import { stream } from "hono/streaming";
import { zValidator } from "@hono/zod-validator";
import { newQuoteStream, prepareQuotes, type Quote, type SwapParams } from "@spandex/core";

const app = new Hono();

app.get("/quotes/stream", zValidator("query", querySchema), async (c) => {
  const swap = c.req.valid("query") satisfies SwapParams;
  return stream(c, async (streamWriter) => {
    const prepared = await prepareQuotes<Quote>({
      swap,
      config,
      mapFn: (quote: Quote) => Promise.resolve(quote),
    });
    await streamWriter.pipe(newQuoteStream(prepared));
  });
});
```

Add `/get_quotes` and `/get_quote` routes if you delegate those actions.

## React Streaming and Async Iterators

`useQuotes` defaults to streamed results (`streamResults: true`) and uses TanStack
`streamedQuery`. Internally, quote/simulation promises are exposed as an `AsyncIterable` so each
result can be yielded as soon as it is ready.

This is what enables progressive quote rendering instead of waiting for all providers to finish.
